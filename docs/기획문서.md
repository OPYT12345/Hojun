---

## 1. 기획 배경 및 문제 인식

기존 수업 현장에서는 다음과 같은 문제가 반복되었다.

- **출석 관리의 번거로움:** 매 수업마다 호명하거나 수기 서명으로 시간이 낭비됨
- **과제 제출 분산:** 이메일, 카카오톡, 구글 폼 등 채널이 분산되어 강사의 취합 부담이 큼
- **학생 상태 파악 어려움:** 수업 중 학생이 모르는 개념이 있어도 손을 들기 어려운 심리적 장벽 존재
- **피드백 지연:** 과제 채점 결과를 학생이 인지하기까지 시간이 걸림

이러한 문제를 해결하기 위해 NFC 태그, 실시간 알림, AI 튜터를 결합한 통합 수업 관리 플랫폼을 기획하였다.

---

## 2. 서비스 개요

NFC 스티커를 강의실에 부착하면, 학생이 스마트폰으로 태그하는 것만으로 출석이 자동 처리된다.  
강사는 웹 브라우저에서 강의 자료 배포, 과제 출제 및 채점, Q&A 답변, 공지 발송, 수강생 관리 등 수업 운영 전반을 관리한다.  
학생은 언제 어디서나 과제를 제출하고, AI 튜터에게 질문하며, 강사의 피드백을 실시간으로 받는다.

## 3. 핵심 기능 목록

### 3-1. 출석 관리
| 기능 | 설명 |
|------|------|
| NFC 자동 출석 | 학생이 NFC 스티커 태그 시 즉시 출석 처리 |
| 단축 URL | `/nfc/{lectureId}` 형태로 강의별 고유 URL |
| 출석 현황 조회 | 강사 화면에서 실시간 출석 목록 확인 |
| 개인 출석 이력 | 학생이 자신의 강의별 출석/결석 현황 조회 |

### 3-2. 강의 자료 관리
| 기능 | 설명 |
|------|------|
| 자료 등록 | PDF, PPT, HWP, 영상, 링크 등 다양한 형식 지원 |
| 파일 업로드 | 서버 저장 후 다운로드 링크 제공 |
| 자료 삭제 | 강사 전용 삭제 권한 |

### 3-3. 과제 시스템
| 기능 | 설명 |
|------|------|
| 과제 등록/수정/삭제 | 강사가 제목, 내용, 기간, 첨부자료 설정 |
| 과제 제출/수정 | 학생이 텍스트 + 파일/링크로 제출, 기간 내 수정 가능 |
| 채점 | 강사가 0~100점 + 피드백 입력, 학생에게 즉시 알림 |
| D-day 표시 | 마감 기한 카운트다운 뱃지 |

### 3-4. Q&A 시스템
| 기능 | 설명 |
|------|------|
| 질문 등록 | 학생이 강의별로 질문 게시 |
| 강사 답변 | 강사가 답변 등록 시 학생에게 알림 발송 |
| AI 답변 제안 | 강사 화면에서 AI가 답변 초안 자동 생성 |

### 3-5. 공지 방송
| 기능 | 설명 |
|------|------|
| 공지 등록 | 강사가 제목 + 내용으로 공지 작성 |
| 전체 발송 | 수강생 전원에게 인앱 알림 자동 발송 |
| 공지 열람 | 학생이 강의별 공지 목록 조회 |

### 3-6. AI 튜터 / 분석
| 기능 | 설명 |
|------|------|
| 학생 AI 튜터 | GPT-4o-mini 기반, 강의 관련 질문에 답변 |
| 강사 AI 분석 | 제출물 경향, 질문 패턴, 출석 현황 종합 분석 |
| AI 상담 | 학생 개인 학습 고민 상담 (강사에게 데이터 미전달) |

### 3-7. 인앱 알림
| 기능 | 설명 |
|------|------|
| 벨 아이콘 | 학생 화면 헤더에 미읽음 배지 표시 |
| 알림 유형 3종 | 과제 채점(GRADE), Q&A 답변(QA_ANSWER), 공지(BROADCAST) |
| 강의명 명시 | 모든 알림에 어느 강의의 알림인지 표시 |

### 3-8. 학생 상태 알리기
| 기능 | 설명 |
|------|------|
| 상태 버튼 | 이해함 / 모르겠음 / 잠깐만요 / 화장실 등 |
| 실시간 반영 | 강사 화면에 학생 좌석별 상태 즉시 표시 |

### 3-9. 학생 관리 (강사용)
| 기능 | 설명 |
|------|------|
| 수강생 목록 | 이름·학번·이메일·자리번호 카드 표시 |
| 출석률 색상 코딩 | 80%↑ 녹색 / 60~79% 주황 / 60%↓ 빨강 |
| 출석 이력 상세 | 카드 클릭 시 날짜별 출석/결석 이력 펼치기 |
| 과제 제출 현황 | 수강생별 과제 제출 건수 배지 표시 |

### 3-10. 비밀번호 변경
| 기능 | 설명 |
|------|------|
| 비밀번호 수정 | 로그인된 상태에서 현재 비밀번호 확인 후 변경 |
| 보안 검증 | 8자 이상 요건 검증, BCrypt 해시 저장 |

### 3-11. 라이트 / 다크 테마
| 기능 | 설명 |
|------|------|
| 테마 전환 | 헤더의 세그먼트 버튼으로 라이트/다크 전환 |
| 테마 유지 | localStorage 저장으로 페이지 이동 후에도 유지 |
| FOUC 방지 | 페이지 로드 즉시 저장된 테마 적용 (깜빡임 없음) |

## 4. 기술 스택

| 분류 | 기술 |
|------|------|
| 백엔드 | Spring Boot 3.2, Spring Data JPA |
| 데이터베이스 | MySQL 8.0 |
| 보안 | HTTPS (TLS, 포트 8443), BCrypt 비밀번호 암호화, 세션 기반 인증, Rate Limit 필터 |
| AI | OpenAI GPT-4o-mini API |
| 프론트엔드 | Vanilla HTML / CSS / JavaScript |
| 배포 | Docker Compose (로컬/재현용), Railway (클라우드 라이브) |
| 빌드 | Maven 3.9 |

## 6. 기대 효과

- 출석 처리 시간 **90% 단축** (호명 방식 대비 NFC 태그 1초)
- 과제 제출/피드백 채널 **단일화** → 강사 업무 부담 감소
- AI 튜터를 통한 **24시간 학습 지원** → 강사 부재 시에도 학습 연속성 확보
- 학생 상태 실시간 파악 → **수업 참여도 및 이해도 향상**
- 수강생 출석률·과제 현황 통합 관리 → **강사의 학생 파악 효율 향상**
