#  수업 비서 프로그램(AI활용 차세대 교육 솔루션)

강의 중 학생 비서 웹 서비스입니다.  
NFC 출석, AI 튜터, 강의 자료 배포, 실시간 Q&A, 과제 제출 기능등 많은 기능을 제공합니다.

---

## 기술 스택

- **백엔드:** Spring Boot 3.2, Spring Data JPA, Server-Sent Events
- **프론트엔드:** Vanilla HTML/CSS/JS (별도 빌드 없음)
- **데이터베이스:** MySQL 8.x
- **AI:** OpenAI GPT-4o-mini

---

# 면접관님들이 실행을 원하는 경우:

## 1. 오픈 URL로 접속 (가장 권장드리는 방법) : https://hojun-production-6399.up.railway.app 가 저희 팀이 배포한 링크입니다. 이쪽으로 접근해서 테스트 하시는 게 훨씬 편하고 저희 입장에서도 권장드립니다.


## 2. Docker로 시작.

오픈된 서비스가 아닌, 컴퓨터에서 따로 실행하고 싶을 실 경우,

Docker만 설치되어 있으면 DB 포함 모든 환경이 자동으로 구성됩니다.

### 사전 요구사항

| 항목 | 버전 |
|------|------|
| Docker Desktop | 최신 버전 |
| Docker Compose | v2 이상 (Docker Desktop에 포함) |

### 실행

```bash
git clone <저장소 URL>
cd login-backend
```

`.env.example` 파일을 복사해 `.env` 파일을 만들고 API 키를 입력합니다:

```bash
cp .env.example .env
```

`.env` 파일을 열어 아래와 같이 수정합니다:

```
OPENAI_API_KEY=발급받은_OpenAI_API_키
```

> AI 기능(AI 튜터, AI 분석)을 사용하지 않는 경우 API 키 없이도 나머지 기능은 정상 동작합니다.

```bash
docker-compose up --build
```

> 첫 실행 시 이미지 빌드 및 의존성 다운로드로 약 3~5분 소요됩니다.  
> 이후 실행은 `docker-compose up` 으로 즉시 시작됩니다.

### 접속

| 페이지 | URL |
|--------|-----|
| 학생 로그인 | https://localhost:8443/student.html |
| 강사 로그인 | https://localhost:8443/instructor.html |
| 회원가입 | https://localhost:8443/signup.html |

> 자체 서명 인증서 경고 → **고급 → localhost(안전하지 않음)으로 이동** 클릭

### 종료

```bash
docker-compose down        # 컨테이너 종료 (데이터 보존)
docker-compose down -v     # 컨테이너 + DB 데이터 완전 삭제
```

## 주요 기능

- **NFC 출석:** 강의실 자리에 부착된 NFC 태그를 스마트폰으로 찍어 출석 처리, 강사 화면에 실시간 반영
- **시간표:** 자기 강의가 언제, 어디서 하는지 한 눈에 볼 수 있습니다.
- **강의 자료:** 강사가 올리는 PDF/PPT/영상/링크 등록 및 학생 열람/다운로드
- **AI 튜터:** 학생이 수업 중 모르는 내용을 AI에게 질문 (GPT-4o-mini) << *이 질문 사항은 AI가 분석해서 강사한테 간접적으로 전달됩니다.*
- **AI 상담:** 학생이 강사한테 데이터를 제공하지 않고, AI에게 여러가지 상담을 할 수 있습니다.
- **AI 분석:** 강사가 제출물·질문 경향 AI 분석 요청 << * 강사 계정 -> 원하는 강의의 "AI 분석"을 누르면, AI가 학생의 여러 행동을 분석할 수 있습니다. 하지만, 현재 저희가 만든 코드에서는 그냥 일반적인 메뉴얼을 출력하게 되었습니다.(임의로 데이터를 넣기 힘들거 같다는 게 원인입니다.) 만약, 저희 프로그램이 정말 현실에서 쓰인다면, AI가 정말로 분석해서, 강사한테 전달해 주는 방식으로 코드를 수정하면 됩니다.
- **실시간 Q&A:** 학생 질문 등록, 강사 답변
- **과제 제출:** 강사가 할당한 과제를 학생이 제출, 그리고 그걸 강사가 채점 및 피드백 가능
- **각 강의의 자신의 출석 현황 체크 가능:** 학생이 각 강의를 자기가 얼마나 빠졌는지 한 눈에 알 수 있습니다.
- **공지:** 강사가 각 강의 전체 학생에게 메세지를 전달할 수 있으며, 알람이 갑니다.
- **학생 상태 알리기:** 내향인 학생을 위해, 강사에게만 보이는 자기의 상태를 고를 수 있습니다.

# "추가로 드리고 싶은 말"

- **1.** NFC의 경우, NFC 스티커에 https://hojun-production-6399.up.railway.app/nfc/1 를 등록하면, NFC를 폰에 갖다 대면, 출석 체크가 가능합니다. << 주소에서 마지막 부분에 있는 1은 각 강의의 lectureID며, 예시 계정으로 예를 들면, 1은 "데이터베이 설계 및 실습", 2는 "웹 프로그래밍 기초", 3은 "알고리즘 개론"입니다. 각 강의의 lectureId를 적어야만, 그 강의에 출석 체크가 가능합니다.
- **2.** 가끔, 예시 계정에 강의가 아무것도 안 뜰 수 있습니다. 그건, 일시적인 오류거나, 한 컴퓨터에, 한 브라우저에 학생 계정과 강사 계정을 동시에 로그인해서 생긴 쿠키 충돌이므로, 쿠키를 삭제하시거나, 로그아웃 후 다시 로그인하시면 정상 작동합니다.
- **3.** 가끔, 예상치 못 한 오류(예를 들면, 분명히 학생 계정인데 학생 계정이여야만 어떤 기능을 사용할 수 있다고 경고 뜸)가 발생할 수 있는데, 이건 한 컴퓨터, 한 브라우저에서 강사 계정과 학생 계정 둘 다 로그인해서 생긴 쿠키 오류로, 쿠키를 삭제하시고, 다시 로그인 하시면 정상작동합니다. (테스트 결과, 같은 브라우저에서 학생 계정, 강사 계정을 동시에 실행하면 생각보다 오류가 많이 발생합니다.)
- **4.** 만약 정말 현실에서 쓰일 경우, 따로 회원가입 기능은 없어질 예정입니다. 왜냐하면, 아무나 강사 계정을 만들거나 아무나 학생 계정을 추가로 만들면 안 되기 때문입니다. 하지만 지금은, 편의를 위해서 임시로 회원가입 기능을 넣었습니다.
- **5.** 저희가 만든 프로그램은 폰과, 컴퓨터 둘 다 작동합니다.
- **6.** NFC 출석의 경우, 오직 휴대폰으로만 가능합니다.
